I'm creating a reusable AWS ECS infrastructure using Terraform, structured as follows:

1. Persistence folder:
   - Contains resources like ECR, security groups, VPC
   - Separate Terraform state stored in S3
   - Example: persistence/main.tf, persistence/variables.tf

2. Dynamic folder:
   - Environment-specific ECS services (e.g., dev, prod)
   - Cloudflare domain creation
   - ECS cluster configuration
   - Example: dynamic/dev/main.tf, dynamic/prod/main.tf

3. Modules folder:
   - Reusable Terraform modules for common components
   - Example: modules/ecs-service, modules/vpc

4. Root directory:
   - Main configuration files
   - Backend configuration for S3 state storage

This structure promotes code reusability and easier management of different environments.
